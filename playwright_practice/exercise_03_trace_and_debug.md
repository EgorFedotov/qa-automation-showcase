# Упражнение 3 — Trace Viewer и инструменты отладки (день 4)

Playwright даёт инструменты, которым в Selenium нет аналогов. Освой их — на интервью
про это часто спрашивают, а в работе они экономят часы.

## 1. Codegen — запись теста кликами
```bash
playwright codegen https://www.saucedemo.com
```
Открывается браузер + окно с кодом: кликаешь по сайту — Playwright генерирует Python-код
с правильными локаторами. Быстрый старт для нового сценария.

## 2. Trace Viewer — пошаговый разбор прогона
```bash
# прогнать с записью трейса
pytest tests/ui --tracing on

# открыть трейс (путь появится в test-results/)
playwright show-trace test-results/<...>/trace.zip
```
В трейсе: таймлайн действий, DOM-снапшот на каждом шаге, сеть, консоль. Идеально для разбора
«почему упал тест». В нашем проекте скриншот падения уже цепляется в Allure (см. `conftest.py`).

## 3. Смотреть глазами
```bash
pytest tests/ui --headed --slowmo 500     # видимый браузер, замедление 500мс
```

## 4. Кроссбраузерность (день 7)
```bash
pytest tests/ui --browser chromium --browser firefox --browser webkit
```
Тесты прогонятся на всех трёх движках. Полезно показать в резюме/на интервью, что фреймворк
кроссбраузерный.

---

DOM-снапшот до и после клика. Опиши в 1–2 предложениях, что увидел (для себя — закрепить).
